# Phishing Email Investigation (False Positive)

## Overview

This investigation documents the analysis of a phishing email alert generated after an inbound email containing an external hyperlink triggered the organization's email security controls. The investigation included reviewing the email contents, validating the embedded URL using threat intelligence, analyzing SIEM logs for user interaction, and determining whether the alert represented malicious activity.

Determine whether the inbound email represented a legitimate phishing attempt that resulted in user interaction or whether the alert was a false positive.

## Investigation Process

1. Reviewed the phishing alert generated by the email security system.
2. Identified the sender, recipient, subject line, and embedded URL.
3. Queried SIEM logs for activity related to the recipient and embedded domain.
4. Verified the domain using TryDetectThis.
5. Reviewed firewall and proxy logs for evidence of outbound connections.
6. Correlated all available evidence before determining the alert classification.

---

## Findings

The investigation confirmed that the inbound email contained an external hyperlink, which triggered the phishing detection rule.

Threat intelligence analysis classified the domain as **Clean**.

SIEM investigation did not identify any outbound connections to the embedded URL or evidence that the recipient interacted with the email.

No additional suspicious activity or indicators of compromise were identified during the investigation.

---

## Incident Classification

**False Positive**

The alert was generated because the email contained an external hyperlink with characteristics that matched phishing detection logic. However, threat intelligence classified the destination as clean, and no evidence of user interaction or malicious activity was identified.

---

## Recommendations

- Continue monitoring for future activity involving **hrconnex.thm**.
- Encourage users to verify unexpected emails containing external links before interacting with them.
- Maintain current email filtering and phishing detection policies.
- Continue monitoring for repeated emails from the same sender.
